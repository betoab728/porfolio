Sistema de gestión de Centros técnicos

Sistema diseñado para optimizar la administración de centros técnicos dedicados a la reparación y mantenimiento de equipos. Su propósito principal es centralizar y automatizar procesos clave como el registro de clientes, equipos y servicios, la gestión de órdenes de trabajo, el seguimiento del estado de reparaciones y la generación de reportes. El objetivo principal es proporcionar una herramienta eficiente que facilite el control operativo, reduzca tiempos de gestión, y mejore la experiencia tanto del personal técnico como de los clientes.

Tecnologías Utilizadas

Angular

Spring Boot

PostgreSQL

Net 8

Características Principales

  • Gestión de usuarios con autenticación segura
  • Registro de clientes y trabajadores
  • Gestion y monitoreo de órdenes de reparación
  • Reporte en pdf

Retos y Soluciones

Integrar dos frameworks robustos como Spring Boot y Angular representó un reto significativo, ya que ambos añaden complejidad al desarrollo al tiempo que ofrecen ventajas importantes, como modularidad y escalabilidad. La elección de PostgreSQL como base de datos relacional facilitó la integración entre el backend y el frontend, simplificando la gestión de datos. Otro desafío importante fue el despliegue del sistema en un entorno web distribuido; sin embargo, este obstáculo se superó eficientemente mediante el uso de Docker, lo que permitió una implementación más controlada y consistente.
